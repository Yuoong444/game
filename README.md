# 마법봉 강화 RPG v62

모바일/PC에서 바로 실행 가능한 HTML/CSS/JS 기반 마법봉 강화 게임입니다.  
이번 v62는 속성 강화 시스템 추가 및 밝은/다크 테마 전체 정합성 수정 버전입니다.

## 실행 방법

1. ZIP 파일 압축을 풉니다.
2. `index.html` 파일을 브라우저에서 엽니다.
3. GitHub Pages에 올릴 때는 압축을 푼 파일 전체를 저장소 루트에 업로드합니다.

## 파일 구조

```text
magic_wand_rpg_v62/
├─ index.html
├─ style.css
├─ script.js
├─ README.md
└─ img/
   ├─ wand/
   │  └─ by_attribute_stage/
   │     └─ wand_{attribute}_lv{0~7}.png   (7속성 × 8단계 = 56개)
   ├─ wand_e.png ~ wand_sss.png            (구버전 호환용)
   ├─ wand_original_transparent.png
   ├─ button_auto.png
   ├─ button_enhance.png
   ├─ summon_effect.png
   ├─ ui_card_bg.png
   └─ ui_inventory_card.png
```

## v60 주요 개선사항

### 1. README.md 전환

- GitHub에서 바로 보기 좋게 관리할 수 있도록 마크다운 형식으로 정리했습니다.

### 2. 저장 버전 갱신

- 저장 키를 `magicWandRpgSaveV59`로 변경했습니다.
- 기존 `magicWandRpgSaveV58` 데이터도 자동으로 불러오도록 레거시 키에 추가했습니다.
- 저장 데이터 버전은 `59`로 갱신했습니다.

### 3. 골드 밸런스 완화

v58_fixed는 고등급, 특히 SSS 구간에서 클릭 배율이 크게 튈 수 있어 후반 수익이 과하게 증가할 가능성이 있었습니다.

v60에서는 다음 항목을 완화했습니다.

- SSS 기본 클릭 배율 하향
- SSS 레벨별 추가 배율 최대값 완화
- 등급별 기본 골드 보너스 일부 조정
- 등급별 레벨 보너스 일부 조정
- 강화 비용 증가율 소폭 완화
- 자동 수익 구매 비용 증가율 소폭 완화

목표는 초반은 답답하지 않게 유지하면서, 후반 수익 폭발을 줄이는 것입니다.

### 4. 모바일 안정화

- 모바일 하단 강화/마력석 버튼의 안전영역을 보정했습니다.
- 모바일 패널 위치와 높이를 다시 정리했습니다.
- 메뉴 버튼, 테마 버튼이 작은 화면에서 너무 길어지지 않도록 조정했습니다.
- 인벤토리/옵션/도감 영역의 가로 넘침 가능성을 줄였습니다.

### 5. UI 문구 오류 수정

- 채굴 시 실제 성장 EXP는 `+2`인데 메시지는 `+1`로 표시되던 문제를 수정했습니다.
- 성장 훈련 버튼 문구를 실제 보상과 맞췄습니다.
- 마력 주입 버튼 문구를 실제 보상과 맞췄습니다.
- 뽑기 결과 이름 영역이 중복되어 있던 HTML을 정리했습니다.

### 6. 옵션 설명 추가

랜덤 옵션 화면에 간단한 설명을 추가했습니다.

- 클릭/골드: 채굴 수익 증가
- 자동: 마력석 자동 수익 증가
- 보석: 보석 획득 기회 증가
- 성공률: 강화 성공률 보정

### 7. 가방 성능 안정화

- 인벤토리 최대 보관 수를 `500개`에서 `220개`로 조정했습니다.
- 모바일에서 많은 아이템을 렌더링할 때 발생할 수 있는 렉을 줄이기 위한 조정입니다.

## v62_1 변경사항 (mobile_income_fix)

### 1. 모바일 백그라운드 자동 수익 끊김 수정 (핵심 수정)

- **문제:** 모바일 브라우저는 탭이 백그라운드로 전환되면 `setInterval`을 최대 1분 간격으로 throttle합니다. 이로 인해 자동 수익이 사실상 중단되고, 복귀해도 그 시간 동안의 수익이 소멸되었습니다.
- **수정:** `Page Visibility API`(`visibilitychange` 이벤트)를 적용했습니다.
  - 탭이 숨겨질 때(`hidden`) 현재 시각을 기록하고 세이브합니다.
  - 탭이 다시 보일 때(`visible`) 경과 시간을 계산해 빠진 자동 수익 틱을 일괄 지급합니다.
  - 알림 메시지로 지급된 골드와 경과 시간을 안내합니다. (예: `📱 자리 비운 동안 3분 20초치 자동 수익 지급!`)
- **오프라인 보상 상한:** 최대 1시간(3600틱)치까지만 보상합니다. 악용 방지 목적입니다.

### 2. 틱 계산 방식 개선

- `setInterval` 자체도 `lastTickTime` 기반 경과 틱 수 계산 방식으로 변경했습니다.
- 기기 성능이나 브라우저 상태로 인해 interval이 지연될 경우에도 누락된 틱을 보정합니다.

### 3. SAVE_VERSION 갱신

- 저장 버전을 `621`로 갱신했습니다. (이전 v62 저장 데이터는 자동 호환됩니다.)
